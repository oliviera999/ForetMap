02

Le liant : marqueur de carte

Un gl_chapter_marker peut porter une question de l'un ou l'autre set. Le tirage (drawQuestionFromMarker) lit la config puis délègue au bon pool.

gl_chapter_markersporte-question
qcm_categorie_slug · qcm_question_codelegacy · mig. 097 (écologie)
event_config_json.question{ set: biome|lore, mode: fixed|random, pool | fixedQuestionCode }
Voie A — en partie

Marqueur sur la carte

  • L'équipe arrive sur un repère « question » (isQuestionMarker())
  • useGLMarkerArrival appelle POST /api/gl/games/:id/markers/:mid/present-question
  • Tirage + présentation (choix mélangés, presentationToken JWT), chips de glossaire
  • Affichage dans GLQcmPopover
  • Réponse POST …/games/:id/qcm/answer → score, gl_game_events
Voie B — dé / modale

Tirage catalogue

  • GLQcmModal ouvre un tirage libre
  • Écologie GET /api/gl/qcm/draw (biome + catégorie)
  • Lore GET /api/gl/lore/qcm/draw?chapitreSlugs=…
  • Puis …/questions/:code/present
Voie C — hors partie

Réponse directe

  • Présentation via la même route present
  • Réponse POST /api/gl/qcm/questions/:code/answer
  • Bascule auto vers le set lore selon le préfixe du code
🔒
Verrou d'accès — le réglage gameplay.qcm_mj_only (mig. 114) : si activé, seuls MJ/staff présentent et valident les QCM ; les joueurs ne déclenchent plus le popover.
